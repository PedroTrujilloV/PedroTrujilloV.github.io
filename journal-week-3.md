# How do you approach the planning, design, and writing phases differently when you're working alone vs. in a group?

In both cases is necessary a requirements document. This document contains the characteristics of the software that the client wishes, is very important that this document be very clear because this first document defines the size of the project and protect both parts client - (company/developer) of any abuse from each other (more requirements, no paid extra work, characteristics or requirements not implemented, etc). We have to convert it in user histories to define all components of the software, and for each component the necessary task to develop it. Those tasks are the sequence of steps necessary to program, and can be defined for the algorithms sequence for that component of software. Its format can be a check list, also it can change along the development process, adding/updating/removing but not drastic changes.

For team work, is good try to split the work by each member by components of software, relying on the work of each other, to get the expected information (inputs/output) in the integration. Is necessary the constant communication. and the responsibility of the delivery scheduled, or communicate opportunely the possible issues. It it because this is dependent work, this mean that many tasks can depend of the results of your co worker.
